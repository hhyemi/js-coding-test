### :heavy_check_mark: 내가 푼 답안

```javascript
function solution(participant, completion) {
    var answer = '';
    
    completion.forEach(item => {
        participant.splice(participant.indexOf(item),1);
    });
    
    answer = participant[0];
    
    return answer;
}
```
효율성 테스트에서 시간초과로 실패한 코드이다.  
완주하지 못한 선수를 찾아도 for문이 돌아서 효율이 떨어지는 것 같다.  
<br/>

```javascript
function solution(participant, completion) {
    var answer = '';
    
    participant.sort();
    completion.sort();
    
    for(let i in participant){
        if(participant[i] != completion[i]) return participant[i];
    }
}
```
정렬을 시키고 비교하면서 다른게 있으면 바로 return해서 효율을 높혔다.

### :heavy_check_mark: 참고하면 좋은 다른 답안 

Thanks for https://programmers.co.kr/learn/courses/30/lessons/42576/solution_groups?language=javascript

```javascript
const solution = (p, c) => {
    p.sort()
    c.sort()
    while (p.length) {
        let pp = p.pop()
        if (pp !== c.pop()) return pp
    }
}
```
.pop()을 이용하여 한 코드이다.  
풀이법은 동일하다.
